# Маршрутизация каналов на Dante выходы

Скрипт `route_dante_outputs.py` позволяет маршрутизировать каналы пульта Wing на Dante выходы последовательно.

## Использование

```bash
cd backend
python3 route_dante_outputs.py [IP] [START_CHANNEL] [NUM_CHANNELS] [DANTE_START] [CARD_NAME]
```

### Параметры

- `IP` - IP адрес пульта (по умолчанию: `192.168.1.102`)
- `START_CHANNEL` - Начальный канал для маршрутизации (по умолчанию: `1`)
- `NUM_CHANNELS` - Количество каналов (по умолчанию: `24`)
- `DANTE_START` - Начальный Dante выход (по умолчанию: `1`)
- `CARD_NAME` - Имя карты (по умолчанию: `WLIVE`)

### Примеры

Маршрутизация 24 каналов (1-24) на Dante выходы 1-24 из карты WLIVE:
```bash
python3 route_dante_outputs.py 192.168.1.102 1 24 1 WLIVE
```

Маршрутизация каналов 1-24 на Dante выходы 1-24:
```bash
python3 route_dante_outputs.py
```

Маршрутизация каналов 5-12 на Dante выходы 10-17:
```bash
python3 route_dante_outputs.py 192.168.1.102 5 8 10 WLIVE
```

## Как это работает

Скрипт пытается маршрутизировать каналы через несколько методов:

1. **Прямая маршрутизация через output connection** (`/ch/{ch}/out/conn/grp` и `/ch/{ch}/out/conn/out`)
2. **Альтернативный метод через routing matrix** (`/ch/{ch}/out/grp` и `/ch/{ch}/out/out`)
3. **Через sends** (если прямые методы не работают)

## Важные замечания

⚠️ **OSC адреса для Dante routing могут отличаться** в зависимости от версии прошивки Wing и конфигурации.

Если маршрутизация не работает:

1. Проверьте документацию Wing Remote Protocols для точных OSC адресов
2. Убедитесь, что карта WLIVE правильно настроена в пульте
3. Проверьте настройки в Dante Controller
4. Попробуйте разные варианты имени карты: `WLIVE`, `DANTE`, `WLIVE PLAY`, и т.д.

## Проверка результата

После выполнения скрипта:

1. Проверьте настройки на пульте Wing
2. Откройте Dante Controller и проверьте маршрутизацию
3. Убедитесь, что каналы правильно маршрутизированы

## Отладка

Скрипт выводит информацию о текущей маршрутизации первого канала для отладки. Если нужно проверить маршрутизацию других каналов, можно модифицировать функцию `check_current_routing()`.
