# План тестирования Gain Staging с Dante аудиоинтерфейсом

## Обзор

Этот документ описывает план тестирования модуля автоматической регулировки уровней (Gain Staging) с использованием Dante аудиоинтерфейса.

## Требования

### Оборудование
- Dante аудиоинтерфейс (Dante Virtual Soundcard, Dante-enabled устройство)
- Пульт Wing (опционально, для применения корректировок)
- Тестовые аудиосигналы (микрофоны, инструменты, тестовые генераторы)

### Программное обеспечение
- Python 3.9+
- PyAudio
- Установленные зависимости из `requirements.txt`
- Dante Virtual Soundcard или Dante Controller (для настройки маршрутизации)

## Этапы тестирования

### 1. Обнаружение Dante устройств

**Цель:** Убедиться, что система может обнаружить Dante аудиоинтерфейс.

**Шаги:**
1. Установите и настройте Dante Virtual Soundcard или подключите Dante устройство
2. Запустите тестовый скрипт
3. Проверьте, что Dante устройство отображается в списке

**Ожидаемый результат:**
- Dante устройство обнаружено и отображается в списке
- Показано количество доступных каналов
- Показан sample rate устройства

### 2. Проверка доступности каналов

**Цель:** Проверить, что система может получить доступ к каналам Dante устройства.

**Шаги:**
1. Выберите Dante устройство из списка
2. Выберите каналы для тестирования (рекомендуется 2-4 канала)
3. Убедитесь, что каналы доступны

**Ожидаемый результат:**
- Каналы успешно выбраны
- Система может открыть аудиопоток для выбранных каналов

### 3. Запуск анализа Gain Staging

**Цель:** Проверить запуск анализа уровней аудио.

**Шаги:**
1. Настройте целевые параметры (RMS, LUFS)
2. Запустите анализ
3. Подайте тестовый сигнал на выбранные каналы

**Ожидаемый результат:**
- Анализ успешно запускается
- Система начинает получать аудиоданные
- Отображаются обновления уровней в реальном времени

### 4. Проверка измерения уровней

**Цель:** Проверить точность измерения уровней аудио.

**Проверяемые параметры:**
- **RMS (Root Mean Square):** Среднеквадратичное значение уровня
- **Peak:** Пиковый уровень
- **Crest Factor:** Разница между Peak и RMS
- **Integrated LUFS:** Интегрированная громкость по стандарту ITU-R BS.1770-4

**Шаги:**
1. Подайте стабильный тестовый сигнал на каналы
2. Дождитесь стабилизации измерений (минимум 5 секунд)
3. Проверьте значения уровней

**Ожидаемый результат:**
- RMS измеряется корректно
- Peak измеряется корректно
- Crest Factor рассчитывается правильно
- LUFS рассчитывается после накопления достаточного количества данных (минимум 5 секунд)

### 5. Проверка обнаружения bleeding между каналами

**Цель:** Проверить систему обнаружения перетекания сигнала между каналами.

**Шаги:**
1. Подайте сигнал на один канал (доминирующий)
2. Оставьте другие каналы без сигнала или с очень слабым сигналом
3. Проверьте обнаружение bleeding

**Ожидаемый результат:**
- Система обнаруживает высокую корреляцию между каналами
- Применяется коррекция для каналов с bleeding
- Показывается информация о доминирующем канале

### 6. Проверка расчета корректировок TRIM

**Цель:** Проверить правильность расчета корректировок для достижения целевых уровней.

**Шаги:**
1. После завершения анализа проверьте рассчитанные корректировки
2. Сравните измеренные уровни с целевыми
3. Проверьте формулу расчета: `adjustment = target - measured`

**Ожидаемый результат:**
- Корректировки рассчитываются правильно
- Учитываются ограничения по clipping (peak headroom)
- При использовании LUFS учитывается mix ratio между RMS и LUFS

### 7. Применение корректировок к пульту Wing (опционально)

**Цель:** Проверить применение рассчитанных корректировок к пульту.

**Шаги:**
1. Подключите систему к пульту Wing
2. Примените корректировки
3. Проверьте изменения TRIM на пульте

**Ожидаемый результат:**
- Корректировки успешно применяются
- TRIM изменяется на пульте согласно расчетам
- Логируются все изменения

## Использование тестового скрипта

### Базовое использование

```bash
cd backend
python test_gain_staging_dante.py
```

Скрипт интерактивно проведет вас через все этапы тестирования.

### С подключением к Wing

```bash
python test_gain_staging_dante.py --wing-ip 192.168.1.100
```

### С указанием длительности анализа

```bash
python test_gain_staging_dante.py --duration 15.0
```

### С автоматическим применением корректировок

```bash
python test_gain_staging_dante.py --wing-ip 192.168.1.100 --auto-apply
```

### Все параметры

```bash
python test_gain_staging_dante.py \
    --wing-ip 192.168.1.100 \
    --wing-send-port 2222 \
    --wing-receive-port 2223 \
    --duration 10.0 \
    --auto-apply
```

## Параметры конфигурации

Настройки тестирования можно изменить в `config/default_config.json`:

```json
{
  "automation": {
    "auto_gain": {
      "bleeding_rejection": {
        "enabled": true,
        "correlation_threshold": 0.7,
        "level_difference_threshold_db": 8.0
      },
      "signal_history": {
        "enabled": true,
        "max_history_size": 30,
        "min_valid_samples": 5,
        "signal_threshold_db": -50.0
      },
      "lufs_measurement": {
        "enabled": true,
        "window_size_seconds": 15.0,
        "min_samples_for_lufs": 240000,
        "rms_lufs_mix_ratio": 0.7
      }
    }
  }
}
```

## Типичные сценарии тестирования

### Сценарий 1: Одиночный канал

1. Выберите 1 канал Dante
2. Подайте сигнал с известным уровнем (например, -12 dB RMS)
3. Установите целевой уровень -18 dB RMS
4. Проверьте, что корректировка = -6 dB

### Сценарий 2: Множественные каналы с bleeding

1. Выберите 2-4 канала
2. Подайте сильный сигнал на канал 1
3. Оставьте другие каналы без сигнала
4. Проверьте обнаружение bleeding на других каналах

### Сценарий 3: LUFS измерение

1. Выберите канал
2. Подайте музыкальный сигнал (минимум 5 секунд)
3. Проверьте расчет Integrated LUFS
4. Установите целевой LUFS (например, -23 LUFS для broadcast)
5. Проверьте комбинированную корректировку (RMS + LUFS)

### Сценарий 4: Применение к пульту

1. Подключите к Wing
2. Выполните анализ
3. Примените корректировки
4. Проверьте изменения TRIM на пульте
5. Проверьте, что уровни соответствуют целевым

## Устранение неполадок

### Проблема: Dante устройство не обнаружено

**Решения:**
- Убедитесь, что Dante Virtual Soundcard установлен и запущен
- Проверьте настройки аудио в системе
- Перезапустите Dante Virtual Soundcard
- Проверьте, что устройство доступно в системных настройках аудио

### Проблема: Нет сигнала на каналах

**Решения:**
- Проверьте маршрутизацию в Dante Controller
- Убедитесь, что сигнал подается на правильные входы Dante
- Проверьте уровни входных сигналов
- Убедитесь, что каналы не muted в Dante Controller

### Проблема: LUFS не рассчитывается

**Решения:**
- Убедитесь, что анализ длится минимум 5 секунд
- Проверьте, что `pyloudnorm` установлен: `pip install pyloudnorm`
- Убедитесь, что на канале есть достаточный уровень сигнала (> -50 dB)

### Проблема: Bleeding не обнаруживается

**Решения:**
- Проверьте настройки `correlation_threshold` (по умолчанию 0.7)
- Проверьте настройки `level_difference_threshold_db` (по умолчанию 8.0 dB)
- Убедитесь, что разница уровней между каналами достаточна
- Проверьте, что сигналы действительно коррелируют

### Проблема: Не удается подключиться к Wing

**Решения:**
- Проверьте IP адрес пульта
- Убедитесь, что пульт включен и подключен к сети
- Проверьте настройки OSC на пульте
- Проверьте firewall настройки

## Ожидаемые результаты

1. ✓ Dante устройство обнаружено
2. ✓ Каналы успешно выбраны и открыты
3. ✓ Анализ запущен и работает
4. ✓ Уровни измеряются корректно (RMS, Peak, Crest Factor)
5. ✓ LUFS рассчитывается после достаточного времени анализа
