# Соглашения (Code + DSP)

## Code style (Python)

- PEP 8, читаемые имена, без скрытой мутации состояния
- По возможности type hints; особенно на границах модулей
- Async I/O через asyncio; блокирующее — выносить/изолировать
- Не добавлять зависимости без явной необходимости

## Safety / Live sound

- Никогда не повышать уровень без жёстких лимитов.
- При сомнениях — снижать, не повышать.
- Любые автоматизации, которые трогают gain/fader, должны иметь **ceiling** и быстрый путь отката.

## DSP ориентиры (если правим аудио-алгоритмы)

- LUFS: ITU-R BS.1770-4, double gating (absolute -70 LUFS + relative -10 LU)
- True Peak: oversampling (минимум 4x)
- K-weighting: HPF 38 Hz + high-shelf +4 dB @ 1681 Hz (типовой каскад)
- EQ biquads: Audio EQ Cookbook (RBJ)

Примечание: если вы добавляете/меняете DSP-логику, добавьте тесты на синтетических сигналах (sine/pink/impulse).
