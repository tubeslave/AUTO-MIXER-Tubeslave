# Быстрый старт: Тестирование Gain Staging с Dante

## Минимальные требования

1. **Dante Virtual Soundcard** установлен и запущен
2. **Python зависимости** установлены: `pip install -r requirements.txt`
3. **Пульт Wing** (опционально, для применения корректировок)

## Быстрый тест (5 минут)

### Шаг 1: Запуск теста

```bash
cd backend
python test_gain_staging_dante.py
```

### Шаг 2: Следование инструкциям

Скрипт интерактивно проведет вас через:
1. Выбор Dante устройства
2. Выбор каналов (рекомендуется 2-4 канала)
3. Настройку целевых параметров (можно использовать значения по умолчанию)
4. Запуск анализа (минимум 5 секунд)
5. Просмотр результатов

### Шаг 3: Подача тестового сигнала

Во время анализа подайте сигнал на выбранные каналы Dante:
- Микрофоны
- Инструменты
- Тестовый генератор
- Любой аудиоисточник

## Тест с применением к пульту Wing

```bash
python test_gain_staging_dante.py --wing-ip 192.168.1.100
```

После анализа вы сможете применить рассчитанные корректировки TRIM к пульту.

## Рекомендуемые настройки для первого теста

- **Каналы:** 2-4 канала
- **Длительность анализа:** 10 секунд
- **Target RMS:** -18 dB (по умолчанию)
- **Target LUFS:** -23 LUFS (опционально)

## Что проверить

После теста проверьте:

1. ✓ Dante устройство обнаружено
2. ✓ Сигнал обнаружен на каналах
3. ✓ Уровни измеряются (RMS, Peak, Crest Factor)
4. ✓ LUFS рассчитывается (после 5+ секунд)
5. ✓ Корректировки рассчитаны
6. ✓ Корректировки применены (если подключен Wing)

## Типичные проблемы

### "Dante устройство не найдено"
→ Убедитесь, что Dante Virtual Soundcard запущен и виден в системных настройках аудио

### "Нет сигнала на каналах"
→ Проверьте маршрутизацию в Dante Controller и убедитесь, что сигнал подается

### "LUFS не рассчитывается"
→ Анализ должен длиться минимум 5 секунд с достаточным уровнем сигнала
