# Инструкции по запуску тестирования Gain Staging с Dante

## Подготовка

### 1. Установка зависимостей

```bash
cd backend
pip install -r requirements.txt
```

Убедитесь, что установлены:
- `websockets>=11.0` - для WebSocket клиента
- `pyaudio>=0.2.13` - для работы с аудиоустройствами
- `pyloudnorm>=0.1.1` - для расчета LUFS
- `numpy>=1.24.0` - для обработки аудио

### 2. Проверка Dante интерфейса

- Убедитесь, что Dante Virtual Soundcard установлен и запущен
- Или подключено Dante устройство и оно видно в системных настройках аудио
- Проверьте маршрутизацию в Dante Controller (если используется)

### 3. Подготовка пульта Wing (опционально)

- Убедитесь, что пульт включен и подключен к сети
- Запишите IP адрес пульта (по умолчанию `192.168.1.100`)
- Убедитесь, что OSC включен на пульте

## Запуск тестов

### Шаг 1: Запуск backend сервера

В первом терминале:

```bash
cd backend
python server.py
```

Дождитесь сообщения:
```
AutoMixer Server initialized on localhost:8765
Server started on ws://localhost:8765
```

### Шаг 2: Запуск тестового скрипта

Во втором терминале:

#### Базовый тест (без подключения к Wing):

```bash
cd backend
python test_gain_staging_dante.py --duration 10.0
```

#### Тест с подключением к Wing:

```bash
python test_gain_staging_dante.py --wing-ip 192.168.1.100 --duration 10.0
```

#### Тест с автоматическим применением корректировок:

```bash
python test_gain_staging_dante.py --wing-ip 192.168.1.100 --duration 10.0 --auto-apply
```

### Параметры командной строки

- `--backend-url` - URL WebSocket сервера (по умолчанию `ws://localhost:8765`)
- `--wing-ip` - IP адрес пульта Wing (опционально)
- `--duration` - Длительность анализа в секундах (по умолчанию 10.0)
- `--auto-apply` - Автоматически применить корректировки к пульту
- `--results-dir` - Директория для сохранения результатов (по умолчанию `test_results`)

## Что происходит во время теста

1. **Подключение к backend** - установка WebSocket соединения
2. **Обнаружение устройств** - получение списка аудиоустройств
3. **Поиск Dante устройства** - автоматический поиск устройства с "Dante" в названии
4. **Выбор каналов** - автоматический выбор всех доступных каналов Dante
5. **Настройка параметров** - создание настроек для каждого канала
6. **Подключение к Wing** - если указан `--wing-ip`
7. **Запуск анализа** - инициализация Gain Staging анализа
8. **Сбор данных** - сбор измерений в течение указанного времени
9. **Остановка анализа** - завершение анализа и расчет финальных LUFS
10. **Получение результатов** - получение финальных измерений и корректировок
11. **Применение корректировок** - если указан `--auto-apply`
12. **Сохранение результатов** - сохранение всех данных в JSON файл

### Анализ результатов

Для анализа сохраненных результатов используйте:

```bash
python analyze_test_results.py test_results/gain_staging_dante_YYYYMMDD_HHMMSS.json
```

Скрипт выведет:
- Статистику измерений по каналам

- Информацию о bleeding detection
- Финальные корректировки
- Список ошибок

## Проверка функций

Тест проверяет следующие функции:

### ✅ Инициализация
- Успешный старт AudioAnalyzer с Dante устройством
- Открытие аудиопотока для всех каналов

### ✅ Захват аудио
- Получение данных со всех каналов Dante
- Корректная обработка многоканального аудио

### ✅ Расчет уровней
- Корректность RMS измерений
- Корректность Peak измерений
- Расчет Crest Factor

### ✅ Bleeding rejection
- Обнаружение bleeding между каналами
- Коррекция уровней при обнаружении bleeding

### ✅ Signal history
- Накопление истории измерений
- Стабилизация измерений

### ✅ LUFS measurement
- Расчет Integrated LUFS (если включено)
- Накопление достаточного количества данных

### ✅ Adjustment calculation
- Расчет корректировок TRIM
- Учет целевых уровней
- Ограничение по clipping

### ✅ Применение (если подключен Wing)
- Успешное применение TRIM к пульту
- Логирование изменений

## Устранение неполадок

### Backend не запускается

- Проверьте, что порт 8765 свободен
- Проверьте установку зависимостей: `pip install -r requirements.txt`

### Dante устройство не найдено

- Убедитесь, что Dante Virtual Soundcard запущен
- Проверьте системные настройки аудио
- Попробуйте перезапустить Dante Virtual Soundcard

### Нет сигнала на каналах

- Проверьте маршрутизацию в Dante Controller
- Убедитесь, что сигнал подается на правильные входы
- Проверьте уровни входных сигналов

### Не удается подключиться к Wing

- Проверьте IP адрес пульта
- Убедитесь, что пульт включен и подключен к сети
- Проверьте настройки OSC на пульте
- Проверьте firewall настройки

### LUFS не рассчитывается

- Анализ должен длиться минимум 5 секунд
- Убедитесь, что `pyloudnorm` установлен
- Проверьте уровень сигнала (> -50 dB)
