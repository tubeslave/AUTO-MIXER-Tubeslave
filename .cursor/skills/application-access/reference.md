# Справка: доступ к приложениям

## Типичные формулировки для пользователя

**Accessibility (управление другими приложениями):**

> Включите для процесса, который запускает автоматизацию: *System Settings → Privacy & Security → Accessibility* — добавьте **Terminal** (или **Cursor**, или ваш `.app`) и включите переключатель. Полностью закройте и снова откройте это приложение.

**Automation (AppleScript / Scripting Bridge):**

> При первом запуске macOS может спросить разрешение «управлять …». Если окна не было: *Privacy & Security → Automation* — разрешите связку «ваш клиент → целевое приложение».

**Full Disk Access:**

> Нужно только если читаете/пишете в защищённые пути (часть `~/Library`, бэкапы Time Machine и т.п.). Клиент: тот бинарник, который реально открывает файлы (иногда это не IDE, а `python3` из конкретного пути).

## Путаницы, которые стоит снимать

- **Cursor** и **встроенный терминал**: иногда события идут от вложенного shell — пользователю проще выдать права и **Terminal/iTerm**, и **Cursor**, если что-то не срабатывает.
- **Electron dev vs production**: в dev часть API ведёт себя иначе; entitlements в `.app` из `npm run build` смотреть отдельно от `npm start`.

## Связь с этим репозиторием

AUTO-MIXER: backend (Python) и frontend (Electron) обычно **не требуют** macOS Accessibility для OSC/MIDI по сети. Запросы «доступ к приложениям» чаще относятся к **локальной автоматизации** или **системным разрешениям хоста**, а не к протоколу WING/dLive.
