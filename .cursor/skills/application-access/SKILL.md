# Доступ к приложениям (macOS)

## Назначение

Помогать агенту **не предполагать**, что он может управлять или читать другие приложения без явных разрешений ОС, и **корректно направлять пользователя** в System Settings.

## Когда требуется доступ

| Сценарий | Типичное разрешение (macOS) |
|----------|-----------------------------|
| Управление UI чужих приложений (клики, ввод) | **Accessibility** (Универсальный доступ) для клиента, который шлёт события (Terminal, Cursor, скрипт-раннер, ваш .app) |
| AppleScript / «автоматизация» конкретного приложения | **Automation**: разрешить управлять целевым приложением |
| Чтение файлов вне песочницы пользователя | **Full Disk Access** или **Files and Folders** (зависит от пути) |
| Захват экрана / окна другого приложения | **Screen Recording** |
| Микрофон, камера, календарь, контакты | соответствующие пункты в **Privacy & Security** |

Electron/Desktop: дополнительно смотреть **entitlements**, **hardened runtime**, и не смешивать в голове «разрешение в Cursor» с «разрешением у собранного .app».

## Поведение агента

1. **Сначала уточнить механизм**: что именно должно произойти (скрипт, GUI-автоматизация, чтение файла, захват экрана, плагин в DAW и т.д.).
2. **Не обещать** обход TCC или «тихую» выдачу прав без действий пользователя в System Settings (на современных macOS это норма).
3. **Указать конкретного клиента**, которому нужно право (например: *Terminal*, *Cursor*, *Python из …*, *ваш MyApp.app*), а не абстрактно «системе».
4. **Путь в настройках** (Sequoia/Ventura+): *System Settings → Privacy & Security → …* (или локализованные названия).
5. Если задача **не требует** доступа к другим приложениям (только файлы в репозитории, локальный сервер, тесты) — не раздувать инструкции по TCC.

## Мини-чеклист для пользователя

```
- [ ] Определён процесс, которому нужен доступ (Terminal / Cursor / собранное приложение)
- [ ] Выбран правильный пункт Privacy (Accessibility, Automation, Files, Screen Recording, …)
- [ ] После включения перезапущен клиент, если ОС это требует
- [ ] Проверена минимально необходимая операция (одна команда или один сценарий)
```

## Безопасность

- Просить только **минимально необходимые** разрешения под задачу.
- Не советовать отключать **SIP** или глобальные обходы безопасности ради удобства автоматизации.
- Для live sound / продакшена: доступ к микшеру по сети — это **не** TCC «доступ к приложениям»; не путать с OSC/MIDI и сетевыми правилами файрвола.
